**Correzioni, miglioramenti ed elementi obsoleti**

* È stato risolto un problema che impediva, ad alcuni utenti, di salvare l&#39;intero URL durante la mappatura di un URL a un URL di destinazione. (AAM-28510)
* È stato risolto un problema nelle regole di unione di profili per gli utenti RBAC, a causa del quale gli utenti senza l&#39;autorizzazione per visualizzare caratteristiche visualizzavano una pagina vuota invece del messaggio di avviso dell&#39;autorizzazione mancante. (AAM-28361)
* È stato risolto un problema a causa del quale, per alcuni utenti, la pagina Elenco caratteristiche conteneva collegamenti interrotti alle Origini dati. (AAM-29174)
* Abbiamo migliorato la funzionalità Collegamento profilo. La funzionalità ora collega un [UUID Audience Manager](https://marketing.adobe.com/resources/help/en_US/aam/ids-in-aam.html) ad altri tre UUID Audience Manager, per un totale di 4 UUID Audience Manager in un cluster. (AAM-28808)
* Ora esponiamo gli ID del segmento di prova ([SID e csegID](https://marketing.adobe.com/resources/help/en_US/aam/ids-in-aam.html)) con un parametro query testSegments=true nelle seguenti API (AAM-27591):
   * ` .../v1/segments`
   * ` .../v1/destinations/<destinationId>/mappings`

* È stata effettuata la migrazione dei metodi API del segmento a Swagger. Consulta l’articolo sulla [migrazione del codice API di Audience Manager](https://marketing.adobe.com/resources/help/en_US/aam/c_api_swagger.html). (AAM-28601)
* Sono stati rimossi un paio di errori minori nel pubblico indirizzabile per le destinazioni DAID. (AAM-28679)
* È stato risolto un problema nella pagina Amministrazione utente a causa del quale non veniva visualizzato alcun messaggio di errore quando un utente tentava di creare un nuovo utente per una società con un nome utente già in uso. (AAM-28945)
* È stato corretto un errore visivo a causa del quale venivano visualizzate le icone sbagliate nella modalità di ripristino della password, in Audience Marketplace e nell&#39;ordinamento delle colonne in pagine di elenco nell&#39;interfaccia dell&#39;utente. (AAM-28953)
* È stato risolto un problema nell&#39;interfaccia di amministrazione di Audience Manager che impediva l&#39;immissione di ID di utenti finali Audience Marketplace a una società. (AAM-28953)

## [!DNL Social]

**Correzioni**

* Flussi di lavoro di pubblicazione: è stato risolto un problema a causa del quale gli utenti che aggiravano un flusso di lavoro di approvazione ottenevano comunque le notifiche di approvazione. Solamente gli utenti che ricoprono il ruolo di approvatori in un flusso di lavoro riceveranno un messaggio per approvare un post.
* Publish Anywhere: è stato risolto un problema a causa del quale gli utenti non venivano informati quando una foto o un video superavano i limiti di dimensione dell&#39;API delle piattaforme per il caricamento.
* Publish Anywhere: è stato risolto un problema a causa del quale i post di destinazione in un album di foto su Facebook venivano pubblicati senza foto.
* Contenuto Calendario: è stato risolto un problema a causa del quale gli utenti non potevano modificare l&#39;ora di un post figlio pianificato.
* Contenuto Calendario: è stato risolto un problema a causa del quale gli emoticon e il testo non venivano visualizzati in caso di pubblicazione di video su Twitter.
* Rapporti: è stato risolto un problema che causava un errore durante il download del rapporto Panoramica moderazione.
* Dettagli proprietà Facebook: è stato risolto un problema relativo all&#39;acquisizione dei dati sui fan in base alla posizione geografica per le proprietà di Facebook.
* Moderazione: è stato risolto un problema a causa del quale gli utenti non erano in grado di scrivere commenti di più righe per le proprietà o per gli autori.
* Moderazione: è stato risolto un problema a causa del quale la pagina Dettagli autore non mostrava l&#39;intera cronologia delle interazioni con l&#39;autore.
* Proprietà: è stato risolto un problema a causa del quale gli utenti non ricevevano le e-mail di notifica quando le proprietà cessavano di essere autorizzate.
* Tag per social network: è stato risolto un problema a causa del quale alcuni tag applicati ai post di LinkedIn non venivano visualizzati nell’analisi dei post.
